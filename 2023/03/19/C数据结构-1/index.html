​ 鸽了好几天的博客（），现在准备整点花活，开几个大坑，这是其中一个：

Recursion

​ 递归是用来解决那些可以分开治理的问题的好办法。我们分开求解，随后合并，这正是递归的主要思路，也就是说，我们可以把大问题化成性质相同的子问题。我想，代表性的应用正是斐波那契数列的应用

​ 那么，根据这样的递推公式：

long long getNFeb(int pos){
	if(pos == 1 || pos == 2){
        return 1;
    }
    else{
        return getNFeb(pos-1)+getNFeb(pos-2);
    }
}

​ 是的，你拿去验证一下，马上就会发现这个没啥大的问题。但是，现在让你求一下 Feb(50).你就会高兴的发现自己的电脑寄了！因为这样的递归是不恰当的。它大量的调用了函数的，导致频繁的压栈，而这些压栈往往确实不必要的。有大量的值我们已经计算过了，可以使用备忘录的方式直接索引！

long long getFeb(int X){

    // memory Tab
    int x1 = 1;
    int x2 = 1;
    // use 'res' to record the result 
    int res ;
    
    // in this for loop , we specifically use recursion alike to solve problems 
    for(int i = 2; i < X; i++){

        res = x1 + x2;

        x1 = x2;

        x2 = res;

    }

    return res;

}

​ 这样就很快了！

​ 我们再来看一个也是稍微简单但是常见的例子：

​ 我们常常这样书写 pow 函数来求乘方：

long long pow(int X, int N){
    long long res = 1;
    for(int i = 0 ; i< N ; i++){
        res *= X;
    }
    return res;
}

​ 递归如何呢？

long long newPos(int x , int n){
    if(x == 1){
        return X;
    }
    if(n % 2 == 0){
        return newPow(int x,n/2)*newPow(int x,n/2)
    }
    else
    {
        return newPow(int x,n/2)*newPow(int x,n/2)*x;
    }
}

​ 不错，本质上就是二分来求。但是好像不对，我们的递归还是变成了两个部分，似乎还是O( N )不太好，那么，本质上是同样的递归，不妨合并起来： 

long long newPow(int X , int N){
	if(N == 1){
        return X;
    }
    if(N % 2 == 0){
        // 下分为 (X^2)^(N/2)的问题处理，处理的终点是发现 N = 1了的时候停止 
        return newPow(X*X, N/2);
    }
    else
    {
        return newPos(X*X, N/2)*X;
    }
}

​ 很好，这样就是 O(log n)了！

​ 我们再来看一个：

int binarySearch(const int arr[],int findElem,int low,int high){
	if(low > high){
        return -1;// or an abstractions called NOTFOUND as -1;
    }
    
    int mid = low + (high - low)>>1;
    
    if(arr[mid] == findElem){
        return mid;
    }
    
    if(arr[mid] > findElem){
        return binarySearch(arr,findElem,low,mid-1);
    }
    //这里建议写else!否则产生错误，就是递归结束向上返回的过程中又一次进入到这里来导致错误的结果
    else{
        return binarySearch(arr,findElem,mid+1 ,high);
    }
}

​ 这就是二分法的递归写法！

​ 总而言之，这就是递归带给我们程序的小小自信！
