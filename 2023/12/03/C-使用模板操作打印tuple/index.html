C++使用模板操作打印tuple

​ 我们知道tuple是一个变参数模板的类，事实上，tuple的由来很简单：

class tuple : public _Tuple_impl<0, _Elements...>
{
	...;
}

​ 我们先不在乎作为接口分离的tuple_Impl类里头具体是啥，但毫无疑问，他一定变参数。这就促使我们想到，如果我们想要打印，就需要考虑模板递归实现。

​ 我们这次考虑到IO流的对象，故这一次我们将会封装一个方法类：

class tuple_printer

​ 这个类将会担负起来我们的打印行动。老规矩，既然是模板递归，且需要正向打印，就采用参数个数控制打印次数

template<int maxCount, int curSize, typename ... ArgsPrint>
class tuple_printer{
public:
    static void print_tuple(std::ostream& os, std::tuple<ArgsPrint...> tp)
    {
        os << std::get<curSize>(tp) << ", ";
        tuple_printer<maxCount, curSize + 1, ArgsPrint...>::print_tuple(os, tp);
    }
};
  1. 为什么是static？

理由是：方便我们可以直接调用而不需要浪费栈空间生成一个对象来调用函数，换而言之，我们只是想调用一个类下的方法而不关心类本身。静态函数是这样访问的：

tuple_printer<maxCount, curSize + 1, ArgsPrint...>::print_tuple(os, tp);

  1. get方法是…

    可以理解为给出index从而根据地址偏移取到元素。比如说，，对于tuple<int, double, std::string> tp，对之使用get方法（给出一个curPlace参数，比如说2）之后，，我们就取到了这个tuple里的double参数

​ 我们很自然的得到一个疑问——递归函数需要要一个重点，模板递归如何结束呢？答案是——偏特化的类的方法来结束递归

// Partial-initialization class to prevent the unstoppable printing
template<int maxCount, typename ... ArgsPrint>
class tuple_printer<maxCount, maxCount, ArgsPrint...>{
public:
    static void print_tuple(std::ostream& os, std::tuple<ArgsPrint...> tp)
    {}
};

​ 在这个函数里，我们什么也不做（此时此刻tuple里应当什么也没有了，无法再取出元素，这里的声明更加像是为了保证函数是重载的占位符）。这样编译的时候就不会检查出存在潜在的栈溢出的错误。

​ 下面的部分则是对更加方便的接口的包装，同本博客的重点就不是一致的了！

template<typename ... T>
void Print_Tuple(const std::tuple<T...>& t,std::ostream& os = std::cout)
{
    os << "[";
    tuple_printer<sizeof ...(T), 0, T...>::print_tuple(os, t);
    os << "\b\b]";
}

template<typename ... T>
std::ostream& operator<<(std::ostream& os, const std::tuple<T...>& t)
{
    Print_Tuple(t, os);
    return os;
}


int main()
{
    std::tuple<int, double, std::string> t(1, 2, "CharlieChen");
    Print_Tuple(t);
    std::cout << t << t;
}
